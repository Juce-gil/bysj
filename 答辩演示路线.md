## 四、老师可能会问的问题与建议回答

### Q1：为什么没有做线上支付？
建议回答：
> 本系统定位为校园二手信息发布与线下面交平台，主要解决信息匹配与预约流程问题，因此未设计线上支付，避免引入更复杂的资金安全与支付合规问题。

### Q2：为什么联系方式不是完全公开？
建议回答：
> 为了保护用户隐私，联系方式在业务流程中做了适度控制，只在特定场景或登录条件下展示。

### Q3：数据库是否已经设计？
建议回答：
> 数据库表结构已经完成设计，包含用户、商品、求购、预约、评论、收藏、通知、公告等核心表。当前主线默认运行在 MySQL 持久化模式，重启后动态数据仍会保留，同时也保留了 memory profile 作为纯演示备用方案。

### Q4：项目还有哪些不足？
建议回答：
> 当前项目主要不足在于自动化测试覆盖面还可以继续扩展，后台细节体验和安全性还可以进一步增强。但核心业务流程已经完整实现，并且主线数据已经支持 MySQL 持久化。
